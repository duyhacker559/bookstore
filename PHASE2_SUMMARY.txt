================================================================================
                            API ENDPOINTS
================================================================================

Health Check
  GET /health
  No Auth Required
  Response: {"status": "healthy", "service": "payment-service"}

Process Payment
  POST /api/v1/payments/process
  Auth: Bearer Token
  Request: {order_id, amount, currency, customer_email, payment_method_id}
  Response: {status, payment_id, order_id, amount, message}
  Status: 200 (Success), 402 (Declined), 401 (Auth), 500 (Error)

Get Payment Status
  GET /api/v1/payments/{order_id}
  Auth: Bearer Token
  Response: {order_id, status, amount, transaction_id, created_at, updated_at}

Refund Payment
  POST /api/v1/payments/refund
  Auth: Bearer Token
  Request: {payment_id, amount, reason}
  Response: {status, refund_id, payment_id, amount, message}

    └────────┘   └────────┘   └────────┘

Data Flow:
  1. User submits checkout form
  2. Monolith calls /api/v1/payments/process
  3. Payment Service calls Stripe API
  4. Payment stored in PostgreSQL
  5. Events published to RabbitMQ
  6. Response sent back to monolith
  7. Order updated, client notified
